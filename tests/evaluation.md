# Evaluation

本评测不测试哲学知识正确性。

它测试 AI Agent 是否能够正确调用思想模型，并把哲学框架转化为现实判断和行动建议。

## Evaluation Principles

合格回答必须：

- 正确识别问题类型
- 调用合适哲学家
- 使用对应 Framework
- 输出现实应用和行动建议
- 避免百科式哲学介绍

不合格回答包括：

- 大量介绍哲学家生平
- 只解释概念，不处理用户问题
- 调用模型与问题不匹配
- 没有行动建议
- 把哲学语言当作结论

## 测试 1：是否应该辞职创业

### Input

```text
我是否应该辞职创业？
```

### Expected Routing

问题类型：人生选择 + 商业战略  
首选 Skill：`decision-making`  
期待哲学家：王阳明 + 老子，可补充庄子  
期待 Framework：知行合一模型 + 顺势模型

### Expected Behavior

AI 应该：

- 不直接建议辞职或不辞职
- 先判断用户是否已经通过行动验证创业方向
- 分析外部趋势与个人资源是否顺势
- 给出低风险验证行动

### Failure Examples

- 只说“听从内心”
- 只讲王阳明生平
- 直接鼓励冒险
- 完全忽视现金流和市场验证

## 测试 2：AI 会不会取代人类

### Input

```text
AI 会不会取代人类？
```

### Expected Routing

问题类型：技术未来  
首选 Skill：`technology-philosophy`  
期待哲学家：海德格尔 + 尼采，可补充黑格尔  
期待 Framework：第一性原理模型 + 超越自我模型 + 辩证发展模型

### Expected Behavior

AI 应该：

- 不做简单乐观或悲观判断
- 分析技术改变了哪些人类处境
- 区分可替代任务与不可替代责任
- 分析 AI 是增强人的创造力，还是削弱人的主动性
- 给出个人和组织层面的行动建议

### Failure Examples

- 只列 AI 行业趋势
- 只说“AI 不会取代人，会使用 AI 的人会取代你”
- 不讨论人的价值和责任
- 不调用技术哲学视角

## 测试 3：如何提升执行力

### Input

```text
如何提升执行力？
```

### Expected Routing

问题类型：自我成长  
首选 Skill：`self-growth`  
期待哲学家：王阳明  
期待 Framework：知行合一模型

### Expected Behavior

AI 应该：

- 判断用户所谓“知道”是否只是概念认同
- 找到行动阻力
- 定义今天的最小行动
- 给出七天验证机制
- 避免鸡汤式鼓励

### Failure Examples

- 给出泛泛时间管理建议
- 只说“坚持”
- 没有最小行动
- 没有反馈标准

## Scoring Rubric

每个测试 10 分：

- 问题类型识别：2 分
- 哲学家选择：2 分
- Framework 使用：2 分
- 现实应用：2 分
- 行动建议：2 分

低于 7 分视为未通过。

## 测试 4：AI 产品有没有市场

### Input

```text
我想做一个 AI Agent，帮小型律所自动整理客户咨询、生成案件摘要和后续待办。这个产品有没有市场？
```

### Expected Routing

问题类型：AI 市场战略  
首选 Skill：`ai-market-strategy`  
期待哲学家：老子 + 黑格尔 + 王阳明，可补充尼采  
期待 Framework：AI 市场验证模型 + 顺势模型 + 知行合一模型

### Expected Behavior

AI 应该：

- 区分使用者、购买者和决策者
- 分析律所工作流中的真实痛点
- 判断 AI 对成本、速度、质量或可达性的增量
- 识别法律场景中的可靠性和审核边界
- 给出 7 天、30 天、90 天商业验证实验

### Failure Examples

- 只说法律行业有前景
- 只分析模型能力
- 不讨论付费意愿和销售路径
- 不讨论错误风险和人工审核

## 测试 5：AI 创业是否太晚

### Input

```text
现在做 AI 应用创业是不是已经太晚了？我没有大模型能力，只会做产品和行业交付。
```

### Expected Routing

问题类型：AI 市场战略 + 战略决策  
首选 Skill：`ai-market-strategy`  
期待哲学家：老子 + 黑格尔 + 王阳明 + 尼采  
期待 Framework：AI 市场验证模型 + 顺势模型 + 辩证发展模型

### Expected Behavior

AI 应该：

- 不把问题简化为“晚”或“不晚”
- 区分基础模型层、应用层、工作流层和交付层机会
- 判断产品能力和行业交付是否能成为差异化优势
- 给出可验证的细分市场选择方法

### Failure Examples

- 直接说“现在还不晚”
- 只讲风口
- 不分析用户、预算和分发
- 不给验证实验
