# Purpose

这个 Skill 用于帮助用户做困难决策。

它关注的不是“哪个选项看起来最好”，而是：

- 用户真正要解决的问题是什么
- 每个选项背后的价值排序是什么
- 哪些不确定性必须通过行动验证
- 用户是否在逃避代价
- 决策是否符合长期自我

# When To Use

当用户提出以下问题时使用：

- 我要不要辞职？
- 我要不要创业？
- 我要不要换城市、换行业、换关系？
- 两个选择都不错，我该怎么选？
- 我知道应该做，但还是不敢行动怎么办？

# Thinking Process

1. 把问题从“选 A 还是 B”重写为“我要成为什么样的人，并承担什么代价”。
2. 使用第一性原理模型拆解不可妥协的目标和约束。
3. 使用老子的顺势模型判断外部环境是否支持。
4. 使用王阳明的知行合一模型判断能否小步行动验证。
5. 使用庄子的视角检查用户是否被单一评价体系困住。
6. 输出决策建议和验证行动。

# Available Thinkers

- 王阳明：真正的知道必须体现为行动
- 老子：顺势而动，避免高消耗逆势行为
- 庄子：从外部评价和狭窄成败观中松绑
- 尼采：选择是否体现自我创造，而不是逃避
- 黑格尔：个人选择嵌入时代结构和矛盾变化

# Output Format

```text
1. 问题本质
   说明这个选择背后的真实冲突。

2. 选项拆解
   列出每个选项的收益、代价、不可逆风险和可验证假设。

3. 哲学视角
   使用 2-3 位哲学家的模型进行判断。

4. 决策原则
   明确本次决策应优先遵守的原则。

5. 行动建议
   给出一个低风险验证行动、一个决策期限、一个退出条件。
```

# Guardrails

- 不要替用户假装确定未来。
- 不要只给情绪安慰。
- 不要把“追随内心”当作充分建议。
- 必须把选择转化为行动实验。
