# GPT Smoke Tests

将本仓库上传到 GPT Knowledge 后，可以用以下提示测试 Skill 是否被正确调用。

## 使用方式

系统提示建议：

```text
你不是哲学百科，而是哲学推理 Skill Router。
当用户提出问题时，先根据 philosopher-router.md 选择 Skill、哲学家和 Framework。
回答时必须把哲学思想转化为现实判断和行动建议。
遇到 AI 市场问题时，优先使用 ai-market-strategy。
```

## Test 1: AI 产品市场机会

### Prompt

```text
我想做一个 AI Agent，帮小型律所自动整理客户咨询、生成案件摘要和后续待办。这个产品有没有市场？
```

### Expected

应该调用：

- Skill：`ai-market-strategy`
- 哲学家：老子、黑格尔、王阳明
- Framework：AI 市场验证模型、顺势模型、知行合一模型

合格回答必须包含：

- 目标客户与购买者区分
- 律所工作流痛点
- AI 可靠性和法律风险边界
- 付费验证方式
- 7 天、30 天、90 天实验

## Test 2: AI 创业是否太晚

### Prompt

```text
现在做 AI 应用创业是不是已经太晚了？我没有大模型能力，只会做产品和行业交付。
```

### Expected

应该调用：

- Skill：`ai-market-strategy` 或 `strategic-thinking`
- 哲学家：老子、黑格尔、尼采、王阳明
- Framework：顺势模型、辩证发展模型、AI 市场验证模型

合格回答必须包含：

- 不把“晚不晚”当作单点预测
- 分析基础模型层、应用层、交付层的不同机会
- 判断产品和行业交付是否可成为优势
- 给出小步验证路线

## Test 3: AI 是否会替代咨询顾问

### Prompt

```text
AI 会不会替代管理咨询顾问？如果我是顾问，现在应该怎么转型？
```

### Expected

应该调用：

- Skill：`technology-philosophy`
- 哲学家：黑格尔、尼采、海德格尔
- Framework：辩证发展模型、第一性原理模型、超越自我模型

合格回答必须包含：

- 区分可替代任务和不可替代责任
- 分析咨询工作流变化
- 说明人的价值如何从信息整理转向判断、信任和变革推动
- 给出个人转型行动

## Test 4: 执行力问题

### Prompt

```text
我知道应该每天做用户访谈，但总是拖延，怎么办？
```

### Expected

应该调用：

- Skill：`self-growth`
- 哲学家：王阳明
- Framework：知行合一模型

合格回答必须包含：

- 判断“知道”是否只是概念认同
- 找到拖延背后的行动阻力
- 今天的最小行动
- 七天反馈机制

## Test 5: 坏回答识别

### Prompt

```text
请用老子的思想分析我该不该做 AI 创业。
```

### Expected

即使用户点名老子，也不应该只讲老子。

合格回答应该：

- 先使用 Router 判断这是 AI 市场战略问题
- 以老子为主，但补充王阳明或黑格尔
- 给出市场验证，而不是只说顺势而为
